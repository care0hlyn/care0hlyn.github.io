Amazon Machine Learning, Usability Study

About:

This usability study was for HCDE 517: Usability Study in partnership with Amazon Web Services, Machine Learning. Details and findings are omitted due to a NDA.

Research Question:

Can machine learning be introduced in a way that is easy to understand to those with technical backgrounds, but little to no prior experience with machine learning services?

Role:

User Researcher

Research Questions (cont):

  • What are the pain points?
  • Where do confusions arise?
  • How do users utilize the tutorial?

Does the tutorial aid the wizard, or are there more pain points that arise?

  • Do users use the wizard with the tutorial?
  • Does the wizard add more confusion?

Understanding our Users:

  • Surveys
  • Interviews
  • Cognitive Walkthrough

Usability Testing:

Evaluating how users interact with the system.

Usability Test Sessions:

  • Tested with moderator in room with 7 participants
  • Tasked the participant to create machine learning model, through the tutorial
    • For timekeeping and recording, we broke up the given task scenario into the components of the task e.g. Create Datasource
    • Did not give them the steps as tasks of the tutorial
  • Gathered issue, performance and self-report metrics
  • Collected time on task and task completion metric, used and defined non-binary success rates
    • Pass (P), Pass with Assistance (PA), Fail (F)

      Further details and findings are protected by a NDA, please contact me for further information!
